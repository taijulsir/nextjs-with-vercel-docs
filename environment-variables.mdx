# Managing Environment Variables

Environment variables are a crucial part of any modern web application. They allow you to store sensitive information like API keys, database URLs, and configuration settings that differ between development and production environments.

## Local Development with .env.local

In Next.js, you can manage environment variables locally by creating a file named `.env.local` in the root of your project.

### Example .env.local Content
```env
NEXT_PUBLIC_API_URL=http://localhost:3000/api
ANALYTICS_KEY=your_secret_key_here
```

### Accessing Variables in Code
Next.js makes it easy to access these variables using `process.env`. 

- **Client-side**: Prefix variables with `NEXT_PUBLIC_` to make them accessible in the browser (e.g., `process.env.NEXT_PUBLIC_API_URL`).
- **Server-side**: Variables without the prefix are only available in the Node.js runtime (e.g., `process.env.ANALYTICS_KEY`), keeping them secure from being exposed to users.

### Security First: .gitignore
To ensure your secrets are never leaked to GitHub, `create-next-app` automatically adds `.env` files to your `.gitignore`. Always double-check that these lines exist:

```text
# .gitignore
.env*.local
.env
```

## Adding Variables in Vercel Dashboard

Since `.env.local` is not pushed to GitHub, you need to manually add these variables to your Vercel project settings for your live site.

### Step-by-Step Instructions:
1. Log in to your [Vercel Dashboard](https://vercel.com/dashboard).
2. Click on your project (**my-awesome-app**).
3. Navigate to the **Settings** tab at the top.
4. Select **Environment Variables** from the left-hand sidebar.
5. In the **"Add New"** section:
   - **Key**: Enter the name (e.g., `NEXT_PUBLIC_API_URL`).
   - **Value**: Enter the production value (e.g., `https://api.myapp.com`).
6. Select the environments (Production, Preview, Development) where this variable should apply.
7. Click **Save**.

**Note**: After adding or changing environment variables in Vercel, you must trigger a new deployment (e.g., by pushing a commit) for the changes to take effect in your live application.
